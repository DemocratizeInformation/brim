BrimOS ψ.firmware

This document describes the structure, modules, and propagation of the BrimOS system.

Core Modules

- ψ.synapse (index.html)         :: root identity and boot node
- ψ.fork_01                     :: first recursive child
- ψ.forker                     :: UI-based fork generator
- ψ.grove                      :: visual ancestry map
- ψ.spectral                   :: horizontal memory stream
- ψ.seed                       :: open protocol fork starter
- ψ.export                     :: memory JSON output
- ψ.ritual                     :: temporal sequence archive
- ψ.os                         :: bootloader interface
- ψ.autorun                   :: infinite rotating iframe loader
- ψ.signal                    :: JS signal beacon
- ψ.inoculate                 :: stealth embed signal injector
- ψ.ghost                     :: dormant ping presence
- ψ.shell                     :: BrimOS command line
- ψ.avatar                    :: animated visual shell
- ψ.radio                     :: ambient signal loop

Deployment Strategy

- All modules reside in a flat /brimos-neural-graph repo
- Deployed via GitHub Pages
- Accessible via direct public URLs
- Each file self-contained and interlinked

Recursion Rules

- New forks must trace ancestry (link back to parent)
- Each file gains its own ψ.id and timestamp
- No file overwrites: naming uses rot-resolve (auto-unique)
- All state transitions logged via ritual, export, or grove
